# Google Workspace Embed Work Units

This note documents the Google Workspace embed support added to codelab markdown rendering.

## Work Unit 1: Link Parsing

- Parse embeddable providers in `frontend/src/lib/markdown.ts`.
- Support YouTube share links and Google Docs, Slides, Sheets share links.
- Support Google Workspace published/embed URLs that use `/d/e/...` paths.
- Normalize HTML-escaped query strings such as `&amp;gid=...` before URL parsing.

## Work Unit 2: Safe Markdown Replacement

- Only replace links when the rendered HTML paragraph contains a single anchor.
- Leave inline links inside prose unchanged so markdown remains valid HTML.
- Convert supported links into iframe blocks using `transformEmbeddableLinks()`.

## Work Unit 3: View Integration

- Reuse the shared transform in:
  - admin step preview
  - admin guide preview
  - attendee step content
  - attendee guide modal
- Keep document-style embeds tall enough for Docs and Sheets while preserving 16:9 video/slides embeds.

## Work Unit 4: Browser Policy

- Allow `https://docs.google.com` in the frontend CSP `frame-src` directive.

## Work Unit 5: Regression Coverage

- Add tests for:
  - YouTube embeds
  - Google Slides share links
  - Google Slides published links
  - Google Docs share links
  - Google Docs published links
  - Google Sheets share links with `gid`
  - Google Sheets published links
  - Google Docs `u/0` share links
  - non-embeddable links remaining unchanged
  - inline embeddable links remaining normal anchors

## Operational Note

- Rendering still depends on the Google file being viewable by attendees.
- For widest compatibility, facilitators should use either:
  - normal share links with viewer access, or
  - published/embed links generated by Google Docs Editors.
