# Open Codelabs PRD (Product Requirements Document)

작성일: 2026-02-06

## 1. 제품 개요
Open Codelabs는 Google Codelab 스타일의 핸즈온 세션을 쉽게 운영·관리할 수 있도록 설계된 오픈 소스 플랫폼이다. 퍼실리테이터(관리자)와 참가자 역할을 명확히 분리하고, Markdown 기반 콘텐츠 제작과 AI 기반 코드랩 생성, 실시간 상호작용(WebSocket), 제출/평가/수료증 등 워크숍 운영에 필요한 핵심 도구를 제공한다.

## 2. 문제 정의
- 워크숍 운영자는 콘텐츠 제작, 참가자 관리, 실시간 질의응답, 제출물/피드백 수집을 여러 도구로 분산 처리하고 있다.
- Google Codelab 스타일의 구조화된 학습 경험을 자체적으로 구현하기 어렵다.
- 오프라인/로컬 환경과 클라우드 환경을 모두 지원하는 유연한 운영 도구가 부족하다.

## 3. 목표 및 비목표
### 3.1 목표
- 로컬 환경에서도 빠르게 설치·운영 가능한 핸즈온 플랫폼 제공
- 퍼실리테이터와 참가자 각각의 작업 흐름을 단일 UI에서 지원
- Markdown 기반 콘텐츠 제작과 AI 기반 자동 생성으로 제작 비용 절감
- 실시간 상호작용(채팅/DM/도움 요청) 및 진척도 모니터링 제공
- 수료증, 퀴즈, 피드백, 제출물 기능을 통해 워크숍 운영 효율 향상

### 3.2 비목표
- 완전한 LMS(학습 관리 시스템) 수준의 과제 채점/강의 관리 기능
- 엔터프라이즈급 멀티 테넌시 및 복잡한 권한 계층
- 고가용성/대규모 분산 인프라(현재 단일 서버 중심)

## 4. 사용자 및 이해관계자
### 4.1 주요 사용자
- 퍼실리테이터(관리자): 코드랩 작성, 운영, 참가자 관리, 라이브 지원
- 참가자(Attendee): 코드랩 수강, 질문/도움 요청, 제출/피드백

### 4.2 페르소나 예시
- 기술 워크숍 운영자: 빠른 콘텐츠 제작 및 실시간 운영 도구 필요
- 교육 프로그램 담당자: 수료/피드백 수집, 진행 상황 집계 필요
- 참가자: 명확한 단계 안내, 즉각적인 도움 및 소통 필요

## 5. 제품 원칙
- 접근성: Docker 한 줄로 시작할 수 있어야 한다.
- 사용성: Google Codelab 수준의 가독성과 UI 직관성을 유지한다.
- 유연성: 로컬/클라우드 모두에서 운영 가능해야 한다.
- 확장성: SQLite로 시작하되 필요 시 확장 가능하도록 설계한다.

## 6. 범위 및 기능 요구사항
### 6.1 핵심 기능
1. 코드랩 관리
- 코드랩 생성/수정/삭제 (제목, 설명, 작성자, 공개 여부)
- 단계(Step) 작성 및 정렬, 실시간 미리보기
- 코드랩 Import/Export (ZIP 기반 이식)
- 준비 가이드(Guide) 작성 및 배포

2. AI 코드랩 생성
- 일반 모드: 빠른 생성
- 프로 모드: 계획 → 초안 → 리뷰 → 반영
- 생성 결과 비교(diff) 및 단계별 확인

3. 참가자 관리
- 등록(이름+참가 코드), 중복 이름 방지
- 진행 상황 추적(현재 Step)
- 수료 처리 및 인증서 발급/검증

4. 실시간 상호작용
- 전체 채팅 및 1:1 DM
- 도움 요청 큐 및 상태 관리(pending/resolved)
- 실시간 진행 상황 업데이트

5. 평가 및 제출
- 퀴즈(객관식/서술형)
- 제출물 업로드 및 관리
- 피드백 수집(난이도/만족도/자유의견)

### 6.2 관리자(퍼실리테이터) 기능
- 대시보드: 코드랩 목록, 빠른 접근, 상태 요약
- 편집 도구: Step 편집, 실시간 미리보기, 자동 저장
- 참가자 모니터링: 실시간 진행도, 도움 요청 알림
- 채팅 관리: 전체 공지, DM 지원
- 감사 로그: 이벤트 추적 및 필터링

### 6.3 참가자 기능
- 코드랩 탐색 및 등록
- Step 네비게이션(다음/이전/사이드바)
- AI 질문/답변 히스토리
- 도움 요청 및 상태 확인
- 피드백 제출

### 6.4 시스템 기능
- 데이터 영속성 및 자동 저장
- WebSocket 기반 실시간 동기화
- 입력 검증 및 보안(XSS/SQLi 방지, 파일 업로드 검증)
- 역할 기반 접근 제어(관리자/참가자)

## 7. 비기능 요구사항
### 7.1 성능
- API 응답 평균 50ms 수준
- 페이지 로드 2초 이내
- WebSocket 레이턴시 50ms 이내

### 7.2 확장성
- 동시 사용자 100명 안정적 운영
- SQLite 기반 시작, 필요 시 Postgres 확장 가능

### 7.3 보안
- 관리자 인증 및 세션 관리
- 쿠키 보안 옵션(secure/samesite)
- 입력 검증 및 업로드 제한

## 8. 데이터 및 아키텍처 개요
- 프론트엔드: SvelteKit + Vite + Tailwind CSS
- 백엔드: Rust(Axum) + SQLx
- DB: SQLite 기본, 서버리스 모드에서 Firebase/Supabase 지원
- 실시간: WebSocket 기반 채팅/진척도/도움 요청

## 9. 성공 지표 (KPI)
- 코드랩 생성 평균 소요 시간 감소
- 참가자 완료율 및 피드백 회수율
- 도움 요청 처리 평균 응답 시간
- 세션당 활성 사용자 수

## 10. 제약 사항 및 리스크
- 단일 관리자 로그인만 지원
- SQLite 환경의 동시 쓰기 한계
- 로컬 파일 저장소만 기본 지원
- 단일 서버 WebSocket 구조

## 11. 향후 확장 방향
- 다중 관리자 지원
- PostgreSQL/S3/Redis 기반 확장
- 통계 시각화 및 학습 분석
- 멀티 테넌시 지원

## 12. 오픈 이슈 및 검토 필요 사항
- 다중 관리자 권한 체계 설계
- 대규모 워크숍(수백 명 이상) 운영을 위한 확장 전략
- 클라우드/서버리스 모드 기능 범위 정리

## 12.1 사용자 흐름 (User Flow)
### 퍼실리테이터 핵심 흐름
1. 로그인 → 대시보드 진입
2. 코드랩 생성(제목/설명/공개 설정) → Step 작성
3. 준비 가이드/자료 첨부 → 퀴즈/피드백 조건 설정
4. 세션 시작 → 참가자 모니터링(진행도/도움 요청)
5. 채팅/DM 지원 → 도움 요청 해결
6. 제출물/퀴즈 결과 확인 → 수료 처리 및 인증서 발급

### 참가자 핵심 흐름
1. 코드랩 목록 조회 → 코드랩 선택
2. 이름/참가 코드 등록
3. Step 진행(다음/이전/사이드바)
4. 질문/도움 요청 → DM 응답 수신
5. 퀴즈/제출/피드백 완료 → 수료증 확인

## 12.2 화면 구성 (Wireframe 수준 서술)
### 관리자
- 대시보드: 코드랩 카드 목록, 최근 활동, 진행 중 세션 상태
- 코드랩 편집: 좌측 Step 목록, 우측 Markdown 편집/미리보기 분할
- 라이브 운영: 참가자 리스트(진행도 바), 도움 요청 큐, 채팅/DM 패널
- 제출물/퀴즈: 제출 목록 테이블, 상세 확인 패널

### 참가자
- 코드랩 목록: 카드형 리스트, 검색/필터
- 학습 화면: 좌측 Step 네비게이션, 중앙 콘텐츠, 상단 진행도
- 실시간 지원: 도움 요청 버튼, DM 알림 패널
- 완료 화면: 피드백 폼, 수료증 링크

## 12.3 제품 로드맵 (MVP → Phase 2)
### MVP
- 코드랩 CRUD 및 Step 작성
- 참가자 등록 및 진행도 추적
- WebSocket 기반 채팅/도움 요청
- 퀴즈/피드백/수료증 기본 흐름
- Import/Export

### Phase 2
- 다중 관리자 및 권한 체계
- 고가용성/확장(포스트그레스, Redis)
- S3 등 외부 스토리지 연동
- 분석 대시보드 및 통계 시각화

## 12.4 운영/배포 기준 (SLO/모니터링/비용)
### SLO (권장)
- 가용성 99.5% (월 기준)
- p95 API 응답 300ms 이내
- WebSocket 메시지 지연 150ms 이내

### 모니터링
- 애플리케이션: API 오류율, 응답 시간, WebSocket 연결 수
- 인프라: CPU/메모리/디스크 사용률, DB 잠금/대기
- 비즈니스: 활성 참가자 수, 수료율, 도움 요청 해결 시간

### 비용
- 로컬/소규모: 단일 VM 또는 데스크톱 + SQLite
- 중규모: 컨테이너 기반 배포 + 관리형 DB
- 서버리스: Firebase/Supabase 비용 모델 적용 (트래픽/스토리지 기반)

## 12.5 상세 기능 명세 (Functional Specifications)
### 12.5.1 인증/세션
- 관리자 로그인
- 입력: `ADMIN_ID`, `ADMIN_PW`
- 처리: 로그인 성공 시 관리자 세션 발급(JWT/쿠키)
- 출력: 관리자 대시보드 접근 권한
- 예외: 로그인 실패 시 레이트리밋 적용 및 오류 메시지
- 보안: `COOKIE_SECURE`, `COOKIE_SAMESITE`, `AUTH_SECRETS` 설정 반영

### 12.5.2 코드랩 관리
- 코드랩 생성
- 입력: 제목, 설명, 작성자, 공개 여부, 참가 코드, 퀴즈/피드백 요구 여부
- 처리: 고유 ID 생성, 생성 시간 기록
- 출력: 코드랩 상세로 이동
- 예외: 제목 필수, 중복 ID 방지

- 코드랩 수정
- 입력: 메타데이터, 공개 설정, 준비 가이드, 자료 링크/파일
- 처리: 변경 즉시 저장
- 출력: 최신 상태 반영

- 코드랩 삭제
- 입력: 코드랩 ID
- 처리: 관련 Step/참가자/진척/제출/피드백 정리
- 출력: 목록에서 제거

- Import/Export
- 입력: ZIP 업로드(Import), 코드랩 ID(Export)
- 처리: 구조 파싱/검증, Step 자동 생성
- 출력: 신규 코드랩 생성 또는 ZIP 다운로드

### 12.5.3 Step(단계) 관리
- Step 추가/편집/삭제
- 입력: 제목, Markdown 콘텐츠
- 처리: `step_number` 자동 정렬, 일괄 저장
- 출력: 실시간 미리보기 반영
- 예외: 빈 Step 저장 방지(정책 필요)

### 12.5.4 준비 가이드/자료 관리
- 준비 가이드 작성
- 입력: Markdown 또는 AI 생성 결과
- 처리: 가이드 본문 저장, 버전 갱신
- 출력: 참가자 화면 노출

- 자료 등록
- 입력: 링크 또는 파일 업로드
- 처리: 파일 저장/URL 발급
- 출력: 다운로드/열람 링크

### 12.5.5 AI 코드랩 생성
- 일반 모드
- 입력: 소스 코드/설명 텍스트
- 처리: 단계 생성, 요약/검증 절차 생성
- 출력: Step 초안

- 프로 모드
- 입력: 목표/범위/난이도
- 처리: 계획 → 초안 → 리뷰 → 반영
- 출력: 최종 Step, 변경점(diff) 확인
- 예외: 모델 미설정 시 안내 메시지

### 12.5.6 참가자 등록/관리
- 등록
- 입력: 이름, 참가 코드
- 처리: 중복 이름 방지, 고유 ID 발급
- 출력: 코드랩 시작 화면

- 참가자 목록
- 입력: 코드랩 ID
- 처리: 실시간 목록/상태 제공
- 출력: 진행도, 등록 시간, 현재 Step

### 12.5.7 진행 상황(Progress)
- 자동 저장
- 입력: 현재 Step 이동
- 처리: 즉시 저장 및 동기화(WebSocket)
- 출력: 관리자 대시보드 업데이트
- 예외: 오프라인/연결 끊김 시 재연결 후 동기화

### 12.5.8 실시간 채팅/DM
- 전체 채팅
- 입력: 메시지 텍스트
- 처리: WebSocket 브로드캐스트, 히스토리 저장
- 출력: 실시간 메시지 표시

- DM
- 입력: 대상 참가자, 메시지
- 처리: 1:1 전송, 히스토리 저장
- 출력: 알림 및 메시지 표시

### 12.5.9 도움 요청(Help Request)
- 요청 생성
- 입력: 현재 Step, 메시지(선택)
- 처리: 상태 `pending` 생성, 관리자 알림
- 출력: 대기 상태 표시

- 요청 해결
- 입력: 관리자 처리 완료
- 처리: 상태 `resolved`, 해결 시간 기록
- 출력: 참가자에게 해결 알림

### 12.5.10 퀴즈
- 퀴즈 설정
- 입력: 객관식/서술형 문항, 정답, 배점
- 처리: 코드랩별 퀴즈 저장
- 출력: 참가자 완료 시 노출

- 퀴즈 제출
- 입력: 답안
- 처리: 채점/저장(객관식 자동, 서술형 수동 정책)
- 출력: 결과 및 수료 조건 판단

### 12.5.11 피드백
- 피드백 제출
- 입력: 난이도(1-5), 만족도(1-5), 의견
- 처리: 익명 저장 가능
- 출력: 제출 완료 화면

- 피드백 집계
- 입력: 코드랩 ID
- 처리: 평균/분포 계산
- 출력: 관리자 조회 UI

### 12.5.12 제출물

- 입력: 파일
- 처리: 업로드 검증(확장자/크기), 저장
- 출력: 제출 목록 반영

- 제출 검토
- 입력: 관리자 조회
- 처리: 목록/다운로드 제공
- 출력: 제출물 관리 화면

### 12.5.13 수료증
- 수료 조건 판단
- 입력: 퀴즈/피드백 조건 충족 여부

- 출력: 인증서 링크/검증 URL

### 12.5.14 감사 로그
- 이벤트 기록
- 입력: 관리자/참가자 주요 액션

- 출력: 감사 로그 리스트

### 12.5.15 보안/검증

- 출력: 오류 메시지 및 차단

### 12.5.16 서버리스 모드

- 처리: 백엔드 없이 인증/저장소 연동
- 출력: 동일 기능 제공(제한 범위 정의 필요)

## 13. 용어집 (Glossary)
- 코드랩(Codelab): 단계별 학습 가이드. 제목, 설명, 작성자 메타데이터와 Step 목록으로 구성.
- 스텝(Step): 코드랩 내 개별 학습 단계. 순서, 제목, Markdown 콘텐츠 포함.
- 퍼실리테이터(Facilitator): 코드랩을 생성·관리하고 참가자를 지원하는 관리자.
- 참가자(Attendee): 코드랩을 수강하고 학습을 진행하는 사용자.
- 진행 상황(Progress): 참가자의 현재 Step 및 학습 상태. 실시간으로 동기화됨.
- 도움 요청(Help Request): 참가자가 막혔을 때 보내는 지원 요청. 상태(pending/resolved)로 관리.
- DM(Direct Message): 관리자와 참가자 간 1:1 메시지.
- WebSocket: 실시간 채팅/진척도/도움 요청 알림에 사용되는 양방향 통신 프로토콜.
- 준비 가이드(Guide): 워크숍 사전 준비 사항/자료를 안내하는 문서.
- 퀴즈(Quiz): 학습 평가를 위한 객관식/서술형 질문 세트.
- 피드백(Feedback): 참가자가 제출하는 난이도/만족도/의견 데이터.
- 제출물(Submissions): 참가자가 과제 수행 결과로 업로드한 파일.
- 수료증(Certificate): 퀴즈/피드백 조건을 충족한 참가자에게 발급되는 인증서.
- 감사 로그(Audit Log): 시스템 내 주요 이벤트 기록.
- 서버리스 모드: Rust 백엔드 없이 Firebase/Supabase 기반으로 운영하는 구성.
- 코드 서버 워크스페이스(Code Server Workspace): 코드랩별 개발 환경을 제공하는 선택적 워크스페이스.
